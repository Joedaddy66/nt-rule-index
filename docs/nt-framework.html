Strategic Framework for Number Theory Assets

This interactive document outlines a strategic plan to evolve a number theory research asset management system. It begins with an analysis of the existing PowerShell-based Git repository and its Python-driven build process. The core of this analysis is a proposal to upgrade the system to a static website using MkDocs, which provides a more professional, browsable, and scalable knowledge base. The application allows for a dynamic comparison of the two systems and a clear visualization of their respective automation pipelines, culminating in an actionable roadmap for implementation and future development.

Implementation Roadmap & Recommendations

This section outlines the step-by-step plan for implementing the MkDocs upgrade and provides forward-looking recommendations for continued development. Click on each title to expand the details.

Implementation Roadmap ▸
  • Initialize MkDocs: Run `mkdocs new .` to create the project structure.
  • Configure MkDocs: Update `mkdocs.yml` to reflect the desired navigation (`nav` section from `RULE_PACKS.json`), theme, and plugins.
  • Update Builder: Modify `scripts/build_index.py` to output Markdown files compatible with MkDocs, using frontmatter for titles and other metadata.
  • Configure CI/CD: Create a new GitHub Actions workflow to build and deploy the MkDocs site to GitHub Pages.
  • Migrate Content: Convert existing Markdown content and CSV data into the new MkDocs-compatible structure.
  • Test & Deploy: Validate the local build and push the changes to trigger the automated deployment.
Forward-Looking Recommendations ▸
  • Custom Domain: Point a custom domain to the GitHub Pages site for a professional brand identity.
  • Advanced Plugins: Explore plugins like `mkdocs-material-extensions` for advanced features such as social cards, improved search, and code block enhancements.
  • Automated Dependency Updates: Implement a tool like Dependabot to automatically manage and update project dependencies for security and stability.
